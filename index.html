How to Make Your Own Public Web Site

If you are under 18, please ask your parents before setting up your own web site!

This page was made using the exact instructions described here.

Site Setup for Basic HTML/CSS

Here's how you can make your own simple static web site that anyone can browse. Anything you learned about HTML, CSS, and JavaScript will work here. SQL is not supported.

  1. Set up a Dropbox Account. (This is a cloud file storage service that gives you 5GB for free.)
    1. Create an account on Dropbox.com
    2. Install the Dropbox Software on your computer. (This is optional, but it will make editing much easier. It automatically syncs the Dropbox folder on your computer to your Dropbox account. Therefore, you won't have to do any manual file uploads.) Make sure to use the software to sign in with your DropBox account after installing it.
  2. Log in to KISSr using your DropBox account. (This is a web service that uses your Dropbox account for storage.) Create your subdomain of the form: _____.kissr.com where _____ is replaced with a name you choose. Pick something good because you can't easily change it once you start using it. KISSr allows a minimal level of service for free (1000 web requests, that is HTTP GETs, per month).
  3. Download and install Brackets, a free HTML editor that's really good. You might also like Atom. You can use any HTML editor, so feel free to use another one if you can find a better one, but for the rest of this tutorial I will assume you are using Brackets. Your editor doesn't need any special upload capability because you're just going to edit your site locally and Dropbox Sync will automatically upload your changes to your Dropbox storage.
  4. In Brackets, or whatever HTML editor you are using, use File->Open Folder and open up the Dropbox\Apps\KISSr folder on your local hard drive.
  5. Now, in Brackets in the sidebar on the left that says "Working Files" at the top, you will see a domain name like makeyourownsite.kissr.com or whatever you named your domain name when you set up KISSr. Click on that, it will open, and you will see an index.html. Click the index.html to start editing. You can put other html and image files in this directory too.
  6. To upload images, open a File Browser (File Explorer or Finder) window on your computer, go to the Dropbox\Apps\KISSr\yourownsite.kissr.com folder and just drag them in there.
    TIP: I strongly recommend creating a subdirectory called img or something similar and placing all your images in there. That will help keep things from getting so cluttered.
  7. You are now done with setup! Now all you have to do is create your site and tell people about it.

Setup for Running ProcessingJS Programs from Khan Academy

This is not too difficult. Really you just need to use the ProcessingJS Template from the Khan Acdemy web site.

Just copy the necessary parts of the template to your HTML file and replace this entire part:

    // ProgramCodeGoesHere
    fill(255, 255, 0);
    ellipse(200, 200, 200, 200);
    noFill();
    stroke(0, 0, 0);
    strokeWeight(2);
    arc(200, 200, 150, 100, 0, PI);
    fill(0, 0, 0);
    ellipse(250, 200, 10, 10);
    ellipse(153, 200, 10, 10);

With your own code. Put the HTML tag for the canvas wherever you want the program's canvas to appear. There is an example program over to the right so you can see how it works. You can also use View Source in your browser to view the HTML of this page to see how I created it.

You can CSS style the canvas tag to control the placement and margin of the canvas. If you want a different canvas size or frame rate, change the numbers in this part of the code:

    with (processingInstance) {
        size(600, 600);
        frameRate(30);

NOTE: You might need to make some minor code changes to get your code working outside of KA. See here for some differences between KA and the standard ProcessingJS library.

TIP: There is no "oh noes the error buddy" on real web pages! The training wheels are off now and you are in the same realm as professional web developers. To see errors, look for the JavaScript Console in your browser window. On my browser it is under Tools->Web Developer->Web Console.

What do I do next?

Consider Using GitPages

KISSr limits your usage to 1000 page views per month with the free account. You might want to try using GitPages which doesn't have this limitation, but it's a little harder to use because you will need to learn to use Git to upload your files. This isn't too hard, and there are GUI-based Git clients for Windows and Mac.

(Technically you can just upload files to a Git project using your web browser, but this is annoying enough, to me anyway, that I recommend learning to use the windows/mac Gui client provided by GitHub. But, unlike Dropbox, it will not automatically sync files for you! It is still easier than uploading individual files.)

GitPages has the limitation that you can only have one site per user or Git project.

Other Code Practice Sandboxes and Playgrounds

Here are some other sites where you can play around with JavaScript and even try out server-side scripting:

  • HyperDev supports Node.JS and Express.js for server side scripting! (Recommended)
  • Codepen is client-side only but supports Angular.JS, jQuery, and other JS libraries.
  • JSBin is similar to Codepen in capability.
  • JSFiddle is similar to the previous two but has ProcessingJS as a library option.

Experiment With Local Virtual Servers

This is very advanced, so be prepared to learn a lot of stuff if you want to do this.

Once you are familiar with creating simple web sites you might eventually want to get more advanced by using things like SQL and server-side scripting. You can experiment with this on your own computer using free virtual machine software. For that you will need to learn to use the shell command line in a terminal window.

You can download VirtualBox (Virtual Machine emulator) and run a virtual machine image installed with MEAN.JS for example. There are a lot of other virtual machine images that you can experiment with. The LAMP, WAMP (things ending in AMP) images all provide PHP and SQL servers.

Virtual Web Server Hosting

Once you're familiar with virtual servers, you might want to create your own web site based on one. For that you will need a "real" web hosting account that provides a virtual server with "full stack" server-side scripting engines and a database server.

This is very advanced, so you will need to learn a lot about web devlopment before you can make good use of an environment like this. Get comfortable with running a virtual server on your own computer first. Here are some recommendations.

A virtual server hosting account running a LAMP (Linux, Apache, MySQL, PHP) installation will do pretty much anything that one could want, but the server-side scripting requires knowing PHP which can be learned at CodeAcademy or Programmr.

A virtual server hosting account running MEAN.JS provides an environment with MongoDB and Express.js plus Node.JS for server-side scripting. This is a little more sophisticated than the previous option but it entirely uses JavaScript so learning PHP isn't necessary. Learning the JavaScript frameworks (Express.js, Node.js, and Angular.js) is necessary though, and you can learn more about some of those on CodeAcademy. You can experiment with Node.JS and Express.JS at HyperDev.

Keep in mind virtual servers are an advanced topic and you want to be very comfortable with creating static web sites before delving into virtual servers, server-side scripting, and databases! These are hard core professional level environments.
