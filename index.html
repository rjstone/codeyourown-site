How to Make Your Own Public Web Site

If you are under 18, please ask your parents before setting up your own web site!

Introduction

This page was created primarily for people who have learned HTML/CSS and possibly JavaScript+Processing.js on Khan Academy. Users there are constantly asking "how do I create a real web site?" Khan Academy only provides a sandbox for educational purposes, not web site hosting services. This guide will explain how to use other services to create a web site that anyone on the Internet can browse.

What Should I Have Learned Before Proceeding?

You should have completed at least the Intro to HTML/CSS: Making webpages lessons on Khan Academy. This is the absolute minimum level of knowledge required.

These optional lessons will also be useful to you, but it is not necessary to complete them before continuing here:

  • Intro to JS: Drawing & Animation
  • HTML/JS: Making webpages interactive
  • HTML/JS: Making webpages interactive with jQuery

How am I going to do this? How much does it cost?

These instructions will describe how to set up a very simple static web site using the cloud file storage service Dropbox.com and the web hosting service KISSr. This is free for a basic level of service.

Dropbox will give you 2GB of cloud storage for free, and KISSr will let you set up one site that provides 1,000 web requests per month for free. This is plenty to get started with.

This page was set up using the exact instructions described here.

Site Setup for HTML/CSS/JS

Here's how you can make your own simple static web site that anyone can browse. Anything you learned about HTML, CSS, and JavaScript will work in this environment. SQL is not supported.

  1. Create a Dropbox Account. This will provide storage space for your web site, and you can use it for other purposes as well. If you already have a Dropbox account, you can skip this part since you can use your existing Dropbox account with KISSr.
    1. Create an account on Dropbox.com
    2. Install the Dropbox Software on your computer. The Dropbox software automatically syncs the Dropbox folder on your computer to your Dropbox account. Therefore, you won't have to do any manual file uploads. Make sure to use the software to log in with your DropBox account after installing it.
  2. Log in to KISSr using your DropBox account. This will not disclose your Dropbox password to KISSr.
  3. Create your subdomain of the form: yoursitename.kissr.com where yoursitename is replaced with a name you choose. Pick something good because you can't easily change it once you start using it.
  4. Download and install Brackets, a free HTML editor that's really good. You might also like Atom or VS Code. You can use any HTML editor, so feel free to use another one if you can find a better one, but for the rest of this tutorial I will assume you are using one of those. Your editor doesn't need any special upload capability because you're just going to edit your site locally and Dropbox Sync will automatically upload your changes to your Dropbox storage.
  5. In Brackets, or whatever HTML editor you are using, use File->Open Folder and open up the Dropbox\Apps\KISSr folder on your local hard drive.
  6. Now, in Brackets in the sidebar on the left that says "Working Files" at the top, you will see a domain name like yoursitename.kissr.com or whatever you named your domain name when you set up KISSr. Click on that, it will open, and you will see an index.html file. Click the index.html to start editing. You can put other html and image files in this directory too.
  7. To upload images, open a File Browser (File Explorer or Finder) window on your computer, go to the Dropbox\Apps\KISSr\yoursitename.kissr.com folder and just drag them in there.
    TIP: I strongly recommend creating a subdirectory called img or something similar and placing all your images in there. That will help keep things from getting so cluttered.
  8. You are now done with setup! Now all you have to do is create your site, using your HTML and CSS skills, and tell people about it.

Consider Using GitPages

KISSr limits your usage to 1,000 page views per month with the free account. While you can pay them $5/month to remove this restriction and gain the ability to create as many subdomains as you want, you might want to try using GitPages which doesn't have usage limitations.

GitPages is slightly harder to use because you will need to learn to use Git to upload your files.

While you can just upload files to a Git project using your web browser, you're strongly advised to learn how to use Git because this knowledge will come in very handy for the more advanced server hosting environments described below. I recommend learning to at least use the GUI Git client provided by GitHub.

GitPages has the limitation that you can only have one site per user or Git project. Also, everything you upload there is public to the world unless you pay a subscription fee. So be warned about that.

Get a Free Cpanel and LAMP-Based Web Hosting Account

LAMP stands for Linux, Apache, MySQL, and PHP. While PHP is a bit "old school" at this point (it was one of the first server-side scripting languages) it's one of the few languages supported for scripting on free web hosting accounts.

Here are two major web hosting providers that provide free accounts of this type:

  • X10 Hosting
  • 000Webhost.com

There are plenty of others that you can find by searching around. All of them provide PHP and SQL-based database servers. Files are typically uploaded using an FTP client or via your web browser. (Few have Git support.) The only major disadvantage is that the only server-side scripting language they usually support is PHP which you can learn here at tutorialspoint.

You can also use these to serve static HTML pages with no PHP. So they may be a better alternative to using GitPages.

Learn Server-Side Scripting

Learn server-side scripting with Node.js, PHP, or another server-side scripting language. Why? Because for many purposes you need to be able to write server-side code. This is especially true if you need to store information in a database.

You can learn Node.js and Express.js at tutorialspoint.

There are other server-side scripting environments you can use that are based on python, ruby, and other languages. Tutorialspoint has tutorials for all of them.

Other Code Practice Sandboxes and Playgrounds

Here are some sites where you can play around with JavaScript and even experiment with server-side scripting:

  • HyperDev supports Node.JS and Express.js for server side scripting! (Recommended)
  • Codepen is client-side only but supports Angular.JS, jQuery, and other JS libraries.
  • JSBin is similar to Codepen in capability.
  • JSFiddle is similar to the previous two but has Processing.js as a library option.

Expert Difficulty Level

This section is very advanced, so be prepared to learn a lot of stuff if you want to do any of this.

Once you get to this point, you are getting into the realm of professional web application development. You should know what server-side scripting is, what databases are, and have at least tinkered with these things.

Before continuing, you should learn how to use command line interfaces (CLIs), also called command shells. Which one you use will depend on which operating systems you are using. Web servers most frequently use Linux, so you will need to learn that no matter what. Fortunately for Mac users, OS X uses essentially the same command line interface as Linux with a few minor differences. Windows is a little different since PowerShell is now the preferred CLI, but you can install bash or another Unix/Linux style shell in Windows if you want.

Here are some resources that will help:

  • Learn to use the shell command line on CodeAcademy. This tutorial is interactive and will teach you to use Unix/Linux style command line shells even if you know nothing.
  • A Beginner’s Introduction to Windows PowerShell for those using Windows. If you are not using Windows you can skip this.
  • You can skip learning PowerShell by installing Linux bash on Windows after which you will have the same sort of command line interface in Windows that you do elsewhere.

Install Node.js and a Database Server on Your Desktop or Laptop

You can install Node.js on your Windows, Mac, or Linux computer, run applications from the command line, and use your browser to view them. This is useful for developing and testing your code offline until you're ready to deploy (upload) it.

It is also possible to install Apache, PHP, python, ruby, and other languages with web application frameworks on your local computer for purposes of learning and testing.

You can also install a database such as MySQL or PostgreSQL on your local computer. Both of these come with GUI management tools.

  • MySQL
  • PostgreSQL

Get a Free Containerized Cloud Hosting Account at Heroku

Now that you know the CLI and know some things about both server-side scripting and databases, you might want to try out containerized cloud computing services. There is one that allows you to use Node.js and PostgreSQL for free called Heroku and it looks pretty impressive considering what you can do for free.

You will need to know how to use the CLI on your local computer because it's pretty much impossible to use Heroku without the CLI, but they have excellent step-by-step tutorials on how to deploy and manage applications on their service. You will also need to learn to use Git from the command line.

Heroku is a PaaS (Platform as a Service) type service which doesn't offer you true virtual machines like an IaaS (Infrastructure as a Service) type service. So don't expect to be able to upload your own virtual machine images.

Experiment With Local Virtual Servers

You can simulate an IaaS cloud hosting server on your local computer using free virtual machine software. This is helpful because it gets you used to using a virtual server the same way you would using a virtual server hosting service.

Download VirtualBox (Virtual Machine hypervisor) and run a virtual machine image installed with MEAN.JS for example. There are a lot of other virtual machine images that you can experiment with.

Once you have a virtual machine running, you will want to treat it like some virtual machine that's running on an unknown rack mount server in some unknown location where you will never see it, because this is the situation with actual virtual machine hosting services. In other words, you want to treat your VM like a "headless" machine (one with no monitor or keyboard).

To access and control your VM, you will need to use ssh which is a way to get a command line on remote computers and upload files. Hopefully your VM has Git installed on it, and the best practice for uploading changes to a site or web application is to use Git.

Virtual Web Server Hosting

Once you're familiar with virtual machine servers, you might want to create your own web site based on one and possibly run other types of servers on it such as game servers.

This is very advanced, so you will need to learn a lot about system administration and web devlopment before you can make good use of an environment like this. Get comfortable with running a virtual server on your own computer first. Here are some recommendations.

A virtual server hosting account running a LAMP (Linux, Apache, MySQL, PHP) installation will do pretty much anything that one could want, but it isn't much different than the free LAMP-based web hosting accounts mentioned above. The main advantage is that you will have command line access and can do whatever you want with the virtual server.

A virtual server hosting account running MEAN.JS provides an environment with MongoDB and Express.js plus Node.JS for server-side scripting. This is a little more sophisticated than the previous option but it entirely uses JavaScript so learning PHP isn't necessary. Learning the JavaScript frameworks (Express.js, Node.js, and Angular.js) is necessary though, and you can learn more about some of those on CodeAcademy. You can experiment with Node.JS and Express.JS at HyperDev.

Keep in mind virtual servers are an advanced topic and you want to be very comfortable with creating static web sites before delving into virtual servers, server-side scripting, and databases! These are hard core professional level environments.
